# Master Context

> **Read this file first.** It is the canonical entry point for anyone — human or AI — continuing work on Deepak Labs.

## What This Project Is

Deepak Labs is a long-term product: a *Personal Operating System* that unifies research, engineering work, publications, projects, professional experience, a personal AI assistant, and a personal knowledge platform. It is not a conventional portfolio site.

## The Prime Directive

The repository is the source of truth. Read the documentation to understand the project; update the documentation whenever you change the project.

## How to Get Oriented

1. Read this file.
2. Read [`CURRENT_STATE.md`](CURRENT_STATE.md) — where the project stands right now.
3. Read [`ARCHITECTURE.md`](ARCHITECTURE.md) and [`DECISIONS.md`](DECISIONS.md) — how and why things are built.
4. Read the relevant [`../docs/`](../docs) and [`../specs/`](../specs) before making changes.
5. Follow [`../CONTRIBUTING.md`](../CONTRIBUTING.md).

## Rules for AI Assistants

1. Always read `MASTER_CONTEXT.md` first.
2. Always update `CURRENT_STATE.md` after significant work.
3. Always update `AI_HANDOFF.md` before finishing a session.
4. Never remove documentation.
5. Never change architecture without recording the reason in `DECISIONS.md`.
6. Keep all documentation synchronized.
